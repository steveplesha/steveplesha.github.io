Portfolio

StevePlesha.com - This site!

My portfolio site was my first foray in web development. It is all hand-coded HTML and CSS, no templates or site-builders involved here. I've tried using a variety of technologies in the building of my site. To get experience with responsive design, I've built the page with a mobile-first philosophy using media queries to modify layout and add content as screen size gets larger. The site is hosted on GitHub Pages as I wanted to learn Git revision control for the hosting and pushing my work back and forth from home and work for lunch breaks. I've also implemented Twitter Bootstrap for some of the default styling and grid system along with Sass to get some practice with that.

CarlottaRealty.com

This is a site built with WordPress. The client needed an easy-to-use website that could be molded to fit her needs and allow her to make updates whenever needed. WordPress made the most sense as a solution due to its wide variety of themes available, ease of customization, user-friendly interface, and large selection of plugins for added functionality. The site uses a child theme to modify our selected theme. There are a few plugins used that were modified - a mortgage calculator modified match the visuals of the site and make it more semantic and a testimonial plugin modified with a new and mobile-friendly look. I also created custom pages and graphics for the site.

BrewCellar

This is an app I created for a friend with a large collection of craft beer who was looking for a way to keep track of what he had. The site was designed with a mobile-first approach as he would be using the site from his phone the majority of the time. The design was inspired by craft breweries and the chalk boards they use to list their current brews. The project uses Rails for the back end and is hosted on Heroku.

Car Life - Rails

This is an app built using Ruby on Rails and hosted on Heroku. The goal was to come up with a better solution to track what maintenance has been done on my cars and track regular, scheduled maintenance. I'd find myself getting an oil change and being told I needed some system or another flushed. I'd never remember what was already done, so this app is a fix for that.

The site uses Sass and Coffeescript to generate the stylesheets and JavaScript. The responsive nav menu uses CSS and jQuery to achieve the slide out on hover for larger displays and the toggle down on smaller displays. It uses the paperclip gem for uploading image files for vehicles.

Car Life - AngularJS and Firebase

This is another version of the CarLife app built this time on AngularJS with a Firebase backend.

A lot of the difficulty came not with AngularJS but with Firebase and it's built-in data handling functions. It was worth it in the end as I like the instant updates that can come from any device and automatically reflect any changes across all devices accessing the data.
